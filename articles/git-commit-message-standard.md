# Git 提交信息（commit message）规范

<ArticleMeta />

::: tip 小贴士
如果 type 为 `feat` 或 `fix`，则该 commit 将肯定出现在 Change log 之中
:::

```
feat：新功能（feature）
fix：修改缺陷
docs：提交或修改文档
style：格式（不影响代码运行的变动）修改
refactor：重构（即不是新增功能，也不是修改bug的代码变动）
test：增加测试代码
chore：构建过程或辅助工具的变动
```
